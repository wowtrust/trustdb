# TrustDB 官网 V2 设计说明

## 结论

V2 的核心不是放大旧版间距，而是删除重复叙事。首屏、证明模型、证据流转和快速开始各自只承担一个任务，页面可见信息量相较 V1 下降约 40–50%。

原型图：[`generated/trustdb-official-website-ui-v2.png`](generated/trustdb-official-website-ui-v2.png)

## 页面结构

### 1. Hero — PROVABLE

- 建议高度：`100svh`，最小 `820px`。
- 顶部只保留 Logo、产品、文档、证明模型、系统状态。
- 首屏只传达一句话：`让每一个文件，都有可独立验证的证据链。`
- 主操作：`开始验证`；次操作：`阅读 .sproof 格式`。
- 大面积纯黑留白约占首屏 35%，酸性绿证明信号从留白中进入主标题。

### 2. Proof Model — 五级证明，一条证据链

- 建议高度：`86–100svh`，暖白底。
- L1–L5 只显示等级与一个短标签，不在静态页面同时展示五段说明。
- 详细解释由滚动时逐级出现，并在离开当前级别后淡出。
- 这一章节承担页面唯一一次完整的 L1–L5 叙事。

### 3. Evidence Journey — 三个宏观节点

- 建议高度：`110svh`，深黑底并进行短暂固定滚动。
- 仅保留 `文件 → 透明日志 → 外部锚定` 三个节点。
- Merkle、签名和收据作为证明信号内部的变化呈现，不再重复生成五个说明图标。
- 文字限制为一个标题和一行描述。

### 4. Quick Start

- 建议高度：`76–86svh`。
- 左侧只保留价值主张和 GitHub CTA，右侧保留一个终端窗口。
- 不增加特性列表、徽章、统计数据或额外卡片。

### 5. Footer

- 单行结构：Logo、三个链接、系统状态与 `© 2026 TrustDB`。
- 不放置二次 CTA、站点地图卡片或重复产品描述。

## 视觉系统

| Token | Value | Usage |
| --- | --- | --- |
| `--acid` | `#00FF22` | 主 CTA、证明信号、当前状态 |
| `--void` | `#050705` | 主背景 |
| `--paper` | `#F6FFE9` | 证明模型章节 |
| `--graphite` | `#111511` | 终端与局部层级 |
| `--muted` | `rgba(246,255,233,.58)` | 辅助说明 |
| `--hairline` | `rgba(246,255,233,.12)` | 极少量分隔线 |

- Display：`Inter Tight` 或同类窄体无衬线，超大字号、紧字距。
- Body：中性无衬线，中文正文 16–18px。
- Mono：`IBM Plex Mono`，仅用于哈希与终端内容。
- 除 CTA 和终端外不使用容器卡片；章节主要依靠留白和背景切换区分。

## GSAP 动效蓝图

### Hero

1. 导航 `autoAlpha + y` 进入。
2. 证明信号从左上空白区域穿越地貌，使用一个主时间轴控制。
3. `PROVABLE` 通过 `yPercent + autoAlpha` 进入，随后标题和 CTA 以 80ms stagger 出现。
4. 鼠标视差使用 `gsap.quickTo()`，幅度不超过 14px。

### L1–L5

- 使用顶层 `ScrollTrigger` timeline：`pin: true`、`scrub: 1`。
- 绿色轨道按滚动进度推进；每次只强调一个等级。
- 当前等级显示一段补充说明，其余等级保持安静。
- 不在 timeline 的子 tween 上创建额外 ScrollTrigger。

### Evidence Journey

- 固定章节内部的可视层，动画其子元素而不是被固定元素本身。
- 证明粒子由文件聚合为透明日志，再收束至锚点。
- 只使用 `x`、`scale`、`autoAlpha` 和 CSS 变量，避免动画 `width/left/top`。

### Quick Start

- 章节进入后依次显示命令、成功状态、root 与 anchor。
- GitHub CTA 在终端验证完成后出现，不与命令动画争夺注意力。

### 响应式与无障碍

- 小于 `768px` 时取消 pin 和 scrub，改为离散的短进入动画。
- 通过 `gsap.matchMedia()` 处理桌面、移动端和 `prefers-reduced-motion`。
- reduced-motion 模式直接展示最终状态，保留颜色和排版层级。
- Vue 页面动画全部在 `onMounted()` 后、组件根节点作用域的 `gsap.context()` 内创建，并在 `onUnmounted()` 中执行 `ctx.revert()`。

## 实现验收标准

- 任意一个视口高度内最多出现两个主要内容簇。
- L1–L5 不在其他章节完整重复。
- 页面静止时仍有清晰信息层级，动画不是理解内容的唯一方式。
- 1440px 桌面首屏保留至少 30% 的纯视觉呼吸区。
- 移动端没有横向溢出、被裁切标题或强制固定滚动。
- 循环粒子离开视口后暂停，避免后台动画持续占用资源。
