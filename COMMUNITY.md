# TrustDB Community

Welcome. TrustDB is built in public and values reproducible technical discussion.

## Get help or share an idea

- [GitHub Discussions](https://github.com/wowtrust/trustdb/discussions): questions, integration ideas, deployment experience, and show-and-tell.
- [GitHub Issues](https://github.com/wowtrust/trustdb/issues): confirmed bugs and accepted, scoped engineering work.
- [Contribution guide](CONTRIBUTING.md): branch, issue, pull request, test, and review requirements.
- [Security policy](SECURITY.md): private vulnerability reporting and supported-version policy.
- [Roadmap](ROADMAP.md): current product direction and ways to influence it.

Before opening an issue, use Discussions when you are unsure whether behavior is a bug, need deployment help, or want to explore a feature. This keeps the issue tracker actionable.

## First contributions

Look for [`good first issue`](https://github.com/wowtrust/trustdb/issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22) and [`help wanted`](https://github.com/wowtrust/trustdb/issues?q=is%3Aissue+is%3Aopen+label%3A%22help+wanted%22). Documentation, reproducible examples, interoperability fixtures, platform testing, and small SDK improvements are especially useful.

English and Chinese discussion are both welcome. Technical artifacts, identifiers, commands, and error output should remain exact so other contributors can reproduce the result.
